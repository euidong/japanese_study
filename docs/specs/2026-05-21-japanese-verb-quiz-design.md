## 1. 목적

일본어 학습자가 그림과 함께 동사 활용을 반복 학습할 수 있는 4지선다 퀴즈를 만든다.
원본 학습 자료(45개 동사 그림)와 동일한 어휘를 사용하고, 매 세션마다 무작위로 출제한다.

## 2. 범위

- v1: 45개 동사 전부, 단일 SPA, GitHub Pages 정적 배포.
- v1에서 빼는 것 (YAGNI): 로그인/저장, 통계, 다크모드, 효과음, 다국어, 그룹/유형 필터.
  v2 이후 요청 시 추가.

## 3. 기술 스택

- HTML + CSS + Vanilla JS. 번들러/프레임워크 없음.
- 정적 자산만 — GitHub Pages `main` 브랜치 root에서 직접 서빙.
- 한글·히라가나 표시는 시스템 폰트(웹 안전 폰트 fallback)로 충분.

## 5. 데이터 모델

```js
{
  num: 1,
  dict: "かく",
  masu: "かきます",
  meaning: "씁니다",
  group: 2,            // 1 = 一段(る탈락), 2 = 五段(う→い), 3 = 不規則
  image: "images/verb_01.png"
}
```

45개 동사 전체 목록은 `data.js`에 담는다. 모호한 항목(예: 39번 `きる`)은 그림으로
판단해서 결정하고, 결정 사항은 `data.js` 상단 주석에 남긴다.

## 6. 화면 흐름

단일 페이지에서 3개의 `<section>` 중 하나만 보이도록 `hidden` 토글.

### 6.1 시작 화면
- 제목 "일본어 동사 퀴즈"
- 문항 수 선택: 15 / 30 / 45 (라디오, 기본값 15)
- "시작" 버튼

### 6.2 문제 화면
- 상단: 진행도 `3 / 15` + 프로그레스 바, 현재 점수 `정답 2`
- 중앙: 동사 그림 (한 변 ~240px)
- 그림 아래: 보이는 힌트(3개 중 2개) 라벨/값
- 4개 보기 버튼 (2x2 그리드)
- 답 클릭 시: 정답은 초록, 오답은 빨강 + 정답 강조. 1초 후 자동으로 다음 문제. 마지막 문제면 결과 화면.

### 6.3 결과 화면
- 점수 `12 / 15 (80%)`
- 틀린 문제 목록: 그림 thumb + 출제된 형태 + 내가 고른 답 + 정답
- "다시 시작" 버튼 → 시작 화면

## 7. 퀴즈 생성 로직

1. 시작 버튼 클릭 시점:
   - 45개 동사를 Fisher-Yates 셔플 → 앞에서 N개(15/30/45) 선택.
   - 각 동사마다 `[dict, masu, meaning]` 중 하나를 무작위로 `hidden` 필드로 지정.
2. 각 문제의 4지선다 보기:
   - 정답 1개 + 같은 그룹에서 무작위로 뽑은 오답 후보 풀에서 3개. 같은 그룹에 충분치 않으면 다른 그룹에서 보충.
   - 보기 순서는 셔플.
3. 정답 처리:
   - 정답 시 `score++`, 오답 시 `wrong` 배열에 `{num, hiddenField, picked, correct}` push.
4. 마지막 문제까지 끝나면 결과 화면으로 전환.

## 8. UI/스타일

- 모바일 우선. 컨테이너 `max-width: 480px` 정도, 화면 중앙 정렬.
- 색상: 헤더 진한 네이비(#1f4e79), 보기 버튼 회색 테두리 + 호버 시 강조, 정답/오답 색상 명확.
- 폰트: 시스템 sans-serif 스택(`-apple-system, "Noto Sans CJK KR", ...`).

## 9. 배포

1. `main` 브랜치에 초기 커밋 푸시.
2. GitHub Settings → Pages → Source: `Deploy from a branch`, Branch: `main` / `/ (root)`.
3. 활성화 후 1~2분 내 `https://euidong.github.io/japanese_study/` 에서 접근 가능.
4. 향후 빌드 단계가 필요해지면 `.github/workflows/pages.yml`로 전환.

## 10. 검증 체크리스트

- [ ] 시작 화면에서 15/30/45 선택이 동작한다.
- [ ] 출제된 N문항이 데이터셋 N개 이내로 중복 없이 뽑힌다.
- [ ] 보기는 항상 4개, 정답이 포함되어 있다.
- [ ] 정답 시 초록, 오답 시 빨강 + 정답 강조 후 자동 진행.
- [ ] 마지막 문제 후 결과 화면 표시.
- [ ] 다시 시작 시 셔플이 새로 일어난다.
- [ ] 모바일 폭(375px)에서 레이아웃이 깨지지 않는다.
- [ ] 모든 그림이 `images/verb_NN.png`로 로드된다.

## 11. 오픈 이슈

- 그룹 분류가 모호한 동사: 39 `きる` (이미지 기준 着る = 1그룹), 44 `きる` (이미지 기준 切る = 2그룹).
  같은 표기가 두 번 등장하므로 4지선다 오답 후보로 같이 뽑힐 때 혼동 주의 — 같은 표기가 정답·오답에 모두 들어가지 않도록 보기 풀에서 제외.
